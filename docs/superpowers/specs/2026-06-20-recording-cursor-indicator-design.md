# Индикатор записи через курсор мыши

**Дата:** 2026-06-20
**Статус:** дизайн согласован, готов к плану реализации

## Проблема

Состояние записи сейчас отражается только цветом иконки в системном трее
(красный). Это легко не заметить: пользователь часто забывает остановить
запись, и она «висит» включённой по несколько часов. Нужен заметный
индикатор, который всегда перед глазами.

## Решение

Во время записи подменять системный курсор мыши на красный, при остановке —
возвращать обычный. Курсор всегда в поле зрения, поэтому факт идущей записи
становится очевидным.

### Согласованные дизайн-решения

| Вопрос | Решение |
|--------|---------|
| Внешний вид | **Подкрашенная стрелка** — привычная форма курсора, но насыщенно-красная (не мешает работе, но цепляет взгляд). |
| Охват | **Стрелка + текстовый курсор (I-beam)** — покрывает и «отошёл, мышь на рабочем столе», и «мышь над полем ввода». |
| Сброс «зависшего» курсора | **Всегда на старте приложения** — один безусловный сброс при запуске на случай прошлого краха. |
| Рубильник | Конфиг-ключ `ui.recording_cursor` (bool, по умолчанию `true`). Без галочки в UI (YAGNI), правится в yaml. |

## Архитектура

### Почему подмена системного курсора (WinAPI)

Рассмотренные варианты:

- **A. `SetSystemCursor` / `SystemParametersInfo` (выбран).** Один системный
  вызов при старте подменяет курсоры, один при остановке — возвращает.
  **Ноль фоновой нагрузки**: ни таймеров, ни опроса позиции мыши. Чистый
  `ctypes`, новых зависимостей нет.
- **B. Плавающий значок за курсором.** Отдельное окно поверх всего, таймером
  догоняющее мышь. Постоянный опрос позиции = расход ресурсов, лаги, и это не
  настоящий курсор. Отклонён (противоречит требованию «не жрать ресурсы»).
- **C. `QApplication.setOverrideCursor`.** Меняет курсор только над окнами
  самого приложения; диктовка идёт в чужие окна. Бесполезен здесь.

### Новый модуль `src/utils/recording_cursor.py`

Синглтон `RecordingCursor` с доступом через `get_recording_cursor()` — в стиле
мониторов `_ProcessMonitor` / `_VRAMMonitor` из `system_info.py`.

```
class RecordingCursor:
    activate()    # поставить красные курсоры (если ui.recording_cursor=true)
    deactivate()  # вернуть обычные курсоры
    restore()     # безусловный сброс к системным курсорам (страховка на старте)
```

- `activate()`:
  - читает `config.get("ui.recording_cursor", True)`; если выключено — no-op;
  - если уже активен (`_active`) — no-op (идемпотентность);
  - рисует красную стрелку и красный I-beam через `QPainter` (как
    `tray_icon._generate_icon`), размер берёт из
    `GetSystemMetrics(SM_CXCURSOR/SM_CYCURSOR)`, чтобы совпадал с масштабом
    курсора пользователя;
  - конвертирует каждый `QImage` в `HCURSOR` и ставит через `SetSystemCursor`
    на `OCR_NORMAL` (32512) и `OCR_IBEAM` (32513);
  - ставит `_active = True`.
- `deactivate()`:
  - если не активен — no-op;
  - `SystemParametersInfoW(SPI_SETCURSORS, 0, None, 0)` — перезагружает
    **настоящие курсоры пользователя из реестра** (а не дефолтные виндовые: если
    у пользователя своя тема курсоров, она сохранится);
  - ставит `_active = False`.
- `restore()`:
  - безусловный `SystemParametersInfoW(SPI_SETCURSORS, ...)` + `_active = False`.

**Отрисовка курсоров.** Красная стрелка — полигон классической формы указателя,
залит красным с тонким белым контуром (виден на любом фоне), hotspot в острие
(0, 0). Красный I-beam — вертикальная черта с засечками, hotspot по центру.

**QImage → HCURSOR.** Через WinAPI: из 32-битного ARGB `QImage` создаётся
цветной DIB (`CreateBitmap`/`CreateDIBSection`), монохромная AND-маска из нулей
(прозрачность задаётся альфа-каналом цветного битмапа), затем
`CreateIconIndirect` с `ICONINFO{fIcon=FALSE, xHotspot, yHotspot, hbmMask,
hbmColor}`. Промежуточные `HBITMAP` удаляются через `DeleteObject`.
Замечание: `SetSystemCursor` забирает владение переданным `HCURSOR` и
уничтожает его, поэтому на каждый `activate()` курсоры создаются заново
(дёшево) — хендлы не переиспользуются.

### Привязка к жизненному циклу записи

Все переходы состояния записи уже проходят через `tray_icon.set_state(...)` —
рядом с этими вызовами и вешаем управление курсором (единый источник истины,
курсор и трей всегда синхронны):

| Точка | Файл / метод | Действие |
|-------|--------------|----------|
| Старт записи | `recording_session._do_start_recording` (рядом с `set_state(RECORDING)`) | `get_recording_cursor().activate()` |
| Остановка | `recording_session.stop` (рядом с `set_state(READY)`) | `get_recording_cursor().deactivate()` |
| Сбой фоновой загрузки модели | `recording_session.abort_load_failure` | `get_recording_cursor().deactivate()` |
| Старт приложения | `app.initialize` | `get_recording_cursor().restore()` (безусловная страховка) |
| Выход из приложения | `app.quit` | `get_recording_cursor().restore()` |

## Обработка ошибок

Все вызовы WinAPI обёрнуты в `try/except` с логированием: **сбой подмены
курсора никогда не должен ломать запись**. Если WinAPI недоступен или вернул
ошибку — просто пишем warning и продолжаем (запись работает, курсор остаётся
обычным).

На не-Windows платформах модуль деградирует в no-op (защитная проверка
`sys.platform` / отсутствие `ctypes.windll`), чтобы тесты и импорт не падали.

## Стоимость

- Фоновых ресурсов — **ноль** (нет таймеров, нет потоков, нет опроса).
- Новых зависимостей — **ноль** (чистый `ctypes` + уже используемый Qt).
- Файлов-ассетов — **ноль** (курсоры рисуются в коде).
- Объём кода — один небольшой модуль + ~5 строк привязки в существующих
  методах.

## Тестирование

`voicetype/tests/test_recording_cursor.py` с замоканным WinAPI:

- `activate()` ставит оба курсора (`SetSystemCursor` для `OCR_NORMAL` и
  `OCR_IBEAM`) и выставляет `_active`;
- `activate()` уважает рубильник: при `ui.recording_cursor=false` вызовов
  WinAPI нет;
- `deactivate()` вызывает `SystemParametersInfoW(SPI_SETCURSORS, ...)` и
  сбрасывает `_active`;
- идемпотентность: повторный `activate()` / `deactivate()` при том же состоянии
  безопасны;
- исключение из WinAPI **не пробрасывается** наружу (запись не падает).

Визуальная проверка вручную:

1. Старт записи → стрелка и I-beam становятся красными.
2. Стоп → курсоры вернулись к обычным.
3. Убить процесс во время записи → перезапуск приложения сбрасывает курсор.
