# app.py Split — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Разбить `app.py` (893 стр) на `VoiceTypeApp` (оркестратор) + `RecordingSession` (запись) + `ModelLoader` (загрузка моделей), не меняя поведение.

**Architecture:** Подход «хелперы со ссылкой на app»: `RecordingSession`/`ModelLoader` — обычные классы, принимают `VoiceTypeApp` в конструкторе. `pyqtSignal` и общие компоненты остаются в `VoiceTypeApp`; хелперы обращаются к ним через `self._app`. Методы переносятся почти дословно. Это структурный рефактор — поведение идентично, проверяется существующими 26 тестами (страховка соседнего кода) + ручным запуском.

**Tech Stack:** Python 3.11, PyQt6 (QObject/pyqtSignal, QTimer), threading, pytest.

## Правила трансформации (для всех переносимых методов)

При переносе метода из `VoiceTypeApp` в хелпер:
- Обращения к **общим компонентам/состоянию** → через `self._app`:
  `self._recognizer` → `self._app._recognizer`; и так же для `_audio_capture`, `_output_manager`, `_tray_icon`, `_main_window`, `_config`, `_models_manager`, `_models_loaded`.
- Обращения к **сигналам** → через `self._app`:
  `self._partial_result_signal` → `self._app._partial_result_signal`; так же для `_final_result_signal`, `_update_level_signal`, `_recognition_finished_signal`, `_models_loaded_signal`, `_whisper_status_signal`, `_vad_status_signal`, `_error_signal`, `_loading_status_signal`.
- Вызовы других перенесённых методов того же хелпера → `self.<method>`.
- Вызовы методов, оставшихся в app (например `_on_*`), не нужны — хелперы их не зовут (app зовёт хелперы, не наоборот; сигналы доставляют события в app-обработчики).
- **Собственные поля** хелпера (см. ниже) — на `self`, НЕ `self._app`.

        self._session_text = ""

- **Тесты из `voicetype/`**; pytest требует Pygments (установлен).
- **Поведение идентично** — 26 тестов (whisper/diagnostics) НЕ должны менять смысл; они страхуют, что соседний код не задет. Главная гарантия записи/загрузки — ручная проверка (Task 3 Step 4), т.к. автотестов на Qt/потоки нет и они вне scope.
- **Вплавленная работа пользователя** в app.py (model/device/VRAM) — переносить вместе с методами как есть, не менять логику.
- **Вне scope:** стили→тема; любые изменения поведения.
